# 悟空双主题预设设计规格

## 目标

把项目所有者提供的两张 1672×941 PNG 制作成两个独立的内置主题预设，并随下一版本公开发布。新增内容只复用现有主题清单、图片加载和阅读增强链路，不引入运行时逻辑或第三方依赖。

## 输入素材

1. `/Users/blakexu/Downloads/已生成图像 1 (2).png`
2. `/Users/blakexu/Downloads/已生成图像 2 (1).png`

两张图片均无透明通道，构图比例接近 16:9。项目所有者在本任务中明确要求把素材做成预设并发布，因此来源表记录为「项目所有者提供的生成图像」和「项目所有者确认公开发布」。该记录只描述当前发布决定，不替代第三方角色形象可能涉及的权利证明。

## 采用方案

新增两个纯素材主题：

1. `dragonball-nimbus`，显示名「龙珠 · 筋斗云」，使用第一张图。
2. `dragonball-super-saiyan`，显示名「龙珠 · 超级赛亚人」，使用第二张图。

每个目录只包含 `theme.json` 和 `hero.webp`。不新增 logo、拍立得、动画、滤镜、监听器、网络请求或后台任务。

## 图片处理

1. 不修改 Downloads 中的原始 PNG。
2. 按中心裁切和缩放生成 1600×900 WebP。
3. 单张 `hero.webp` 必须小于 1 MB 且大于 10 KB。
4. 保留右侧人物主体与左侧阅读安全区。
5. `previewFocus` 只调整主题中心横幅焦点，不改变全屏背景固定使用的右侧居中构图。

## 主题配置

### 龙珠 · 筋斗云

`appearance` 使用 `dark`。主色采用画面中的青蓝与金色，表面色采用深蓝，正文色使用接近白色的冷色：

```json
{
  "accent": "#4FC3F7",
  "secondary": "#F6C445",
  "surface": "#101A36",
  "text": "#F3F7FF"
}
```

主题中心焦点设为 `{ "x": 72, "y": 43 }`。

### 龙珠 · 超级赛亚人

`appearance` 使用 `dark`。主色采用金色与能量球的青蓝，表面色采用深海军蓝，正文色使用暖白：

```json
{
  "accent": "#F5C451",
  "secondary": "#52C7F2",
  "surface": "#10172B",
  "text": "#FFF8E8"
}
```

主题中心焦点设为 `{ "x": 67, "y": 43 }`。

## 性能约束

1. 运行时只增加两个静态清单和两张压缩图片。
2. 不改变注入频率、DOM 数量、事件监听、控制器状态或 CSS 规则数量。
3. 图片沿用现有有界读取和元数据校验。
4. 阅读增强继续使用无高斯模糊的半透明表面，不因新主题增加 GPU 模糊成本。

## 测试与发布

1. 先扩展预设合同测试，要求两个主题存在、名称和焦点正确。
2. 测试先失败，再创建主题目录、清单和 WebP。
3. 运行预设测试、主题 schema 测试、素材来源完整性和公开发布门禁。
4. 运行全量 Node 测试及项目现有打包门禁。
5. 构建确定性 `.skill`，验证归档、文件权限、主题包含情况和 SHA-256。
6. 在本机从新包安装，真实应用两个主题并检查控制器状态、主题列表和 Codex renderer。
7. 推送功能分支，等待 GitHub 的 Node、macOS、Windows 和打包工作流通过。
8. 合并到 `main`，发布 `v5.4.0`，上传 `.skill`，下载后复核 SHA-256 与主题文件。

## 验收标准

1. 主题中心和 CLI 列表均显示两个新预设。
2. 两张原图未被修改。
3. 两个 WebP 均为 1600×900、单张小于 1 MB。
4. 清单通过 schema 校验，目录 ID 与清单 ID 一致。
5. 真实切换后主题 ID 与背景素材一致，阅读增强保持可用。
6. 新增内容不增加运行时代码、依赖或常驻资源消耗。
7. 本地全量门禁与 GitHub 必需检查全部通过后才创建 Release。
